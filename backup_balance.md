## 📈 1단계 계단 구조

```
계단 1  (0~1시간)    목표: 2서클 해금
계단 2  (1~2시간)    목표: 3~4서클
계단 3  (2~4시간)    목표: 5~6서클
계단 4  (4~8시간)    목표: 7~8서클
계단 5  (8~12시간)   목표: 마인드맵 핵심 + 25층
계단 6  (12~15시간)  목표: 1단계 완성

벽의 성격 (단조로움 방지):
  계단 1: 마나 부족 (기다림)
  계단 2: 집중력 배분 최적화 (전략)
  계단 3: 마인드맵 비용 (자원 관리)
  계단 4: R 재료 필요 (부녀 협동)
  계단 5: 25층 도달 (R 실력)
```

## ⚙️ 현재 CONFIG (tower_main_v1.html 기준, 수식 재설계 필요)

```javascript
// 집중력
START_FOCUS: 5
FOCUS_PER_REBIRTH: 1

// ★ 이해도 시스템 (수식 설계 필요) ★
// 각 서클 이해도 증가속도 = 집중력 + 상위서클 이해도 기여분
// 1서클 이해도 = 마나 생산속도 (직결)
// 기하급수적 폭발 제어 수식 필요
// 현재 미구현 → 밸런서가 설계 후 시스템 담당에게 전달

// ★ 서클 해금 조건 (이해도 기반, 수치 미정) ★
// GRIMOIRE_UNLOCK_THRESHOLD: [?, ?, ?, ?, ?, ?, ?, ?]
// 목표 해금 시점:
//   2서클: ~17분
//   3서클: ~45분
//   4서클: ~1.5시간
//   5서클: ~3시간
//   6서클: ~5.5시간
//   7서클: ~8시간
//   8서클: ~12시간

// ★ 마나캡 (수치 미정 — 최우선 설계 과제) ★
// MANA_CAP: ???
// 안티매터 1e308과 유사한 매우 큰 수
// 1단계에서는 절대 도달 불가
// 2단계 중반~후반에 처음 도달
// 캡 돌파 → 단계 전환 분기점 or 주요 계단 장애물

// 깨달음 (환생 재화, 구 파편 역할)
INSIGHT_ON_REBIRTH: floor(log10(마나 + 1))
// 마나 1,000   → 깨달음 3
// 마나 10,000  → 깨달음 4
// 마나 1e308   → 깨달음 308 (캡 도달 시)

// 집중력 10단위 보너스
FOCUS_10_BONUS: 2.0

// 침공 (2단계~, 수치 미정)
INVASION_VALUES: [미정]

// R 전사
WARRIOR_START: { hp: 100, atk: 10, spd: 3000 }
WARRIOR_DEF: 0, WARRIOR_CRIT: 0, WARRIOR_PEN: 0

// R 몬스터
MONSTER_HP_BASE: 20
MONSTER_ATK_BASE: 3
MONSTER_SPD_BASE: 1500
MONSTER_SCALE: 0.18

// R→N 보상
R_INSIGHT_INTERVAL: 5
R_INSIGHT_BASE: 1.3

// 단계 전환
STAGE2_FLOOR: 25
STAGE2_GRIMOIRE: 8
```

## 🧮 밸런서가 설계해야 할 것 (우선순위 순)

```
1순위 — 이해도 증가 수식
  목표:
    집중력 + 상위 이해도 기여로 기하급수 증가
    but 계단 구조 (급상승→수평→급상승) 유지
    1단계 10~15시간 안에 8서클 해금

  고려사항:
    t초 후 이해도 = f(집중력, 상위서클기여, t)
    이 함수가 너무 빠르면 게임이 순식간에 끝남
    로그 스케일 or 소프트캡 필요

2순위 — 서클 해금 임계값
  각 서클 이해도가 얼마가 되면 다음 서클 열리는지
  목표 시점 역산해서 수치 도출

3순위 — 마나캡
  매우 큰 수 (안티매터 1e308 참고)
  단계 전환과 연결되는 핵심 수치
  캡 도달 시 어떤 메커닉 발동할지도 설계

4순위 — 깨달음 수급 곡선
  1단계 15시간, 환생 20~30회 기준
  마인드맵 노드 총 비용과 맞춰야 함
```
